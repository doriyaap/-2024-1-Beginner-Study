## GIT이란?
#### 버전 관리 및 협업을 위해 사용하는 오픈소스 소프트웨어
#### GIT을 쓰는 이유
#### - 어떤 파일이, 누가, 언제, 어떻게 수정됐는지 확인하기 위해

## 파일의 생명주기
### * untracked
### * tracked
1. unmodified
2. modified
3. staged
* add the file
* edit the file
* stage the file
* remove the file
* commit

### * Working Directory

## GIT의 파일 관리
>  git 저장소 만들기 - git init
> > git으로 관리할 대상 등록 - git add .
> > > 로컬 저장소로 옮기기 - git commit

## Github
### github 쓰는 이유 : 협업한 내용의 공유를 위해!
### 가능한 것
* 이슈 트래킹
* 코드 리뷰
* Github Actions로 CI/CD
* Github Projects로 프로젝트 업무 관리

## Commit Message
### type
1. feat - 새로운 기능 추가한 경우
2. refactor - 기존 코드 개선한 경우
3. fix - 버그를 수정한 경우 
4. chore - 코드 외의 설정을 바꾼 경우
5. docs - 문서화
6. test - 테스트 코드
